## 4. Тест-кейсы

### TC-01: Регистрация плагина на сервере (пункт 5)

1. Разместить jar в каталог плагинов сервера Set10.
2. Перезапустить серверные сервисы Set10 (по вашему регламенту).
3. Открыть UI Set10:
   - раздел “Внешние системы”
   - настройки типов оплат
4. Проверить наличие:
   - внешней системы `SBG Pay (QR-оплата)`
   - типа оплаты с локализованным именем `Оплата через SBG Pay`

Артефакты:
- скриншот страницы “Внешние системы”
- скриншот страницы типа оплаты

### TC-02: Соответствие полей настроек (пункт 6)

Проверить, что в UI внешней системы доступны поля из `metainf.xml`:
- `sbgpay.baseUrl`
- `sbgpay.deviceToken`
- `sbgpay.lang`
- `sbgpay.currency`
- `sbgpay.ttlSeconds`
- `sbgpay.pollDelayMs`
- `sbgpay.pollTimeoutSeconds`
- `sbgpay.sendReceipt`

Артефакты:
- скриншот формы настроек внешней системы целиком

### TC-03: Сохранение настроек в БД сервера (пункт 8)

1. Изменить 2-3 параметра в UI сервера и сохранить.
2. Найти таблицу настроек (если имя таблицы заранее неизвестно):

```sql
SELECT table_schema, table_name
FROM information_schema.columns
WHERE column_name IN ('module_name','plugin_name','property_key','property_value')
GROUP BY table_schema, table_name
HAVING COUNT(DISTINCT column_name) >= 3
ORDER BY table_schema, table_name;
```

3. Проверить значения для `module_name='uz.sbgpay'` и ключей `sbgpay.*`:

```sql
SELECT module_name, plugin_name, property_key, property_value
FROM <CONFIG_TABLE>
WHERE module_name = 'uz.sbgpay'
ORDER BY property_key;
```

Артефакты:
- SQL-выгрузка до/после изменения параметров

### TC-04: Доставка настроек на кассу (пункт 9)

1. После сохранения настроек на сервере дождаться транспорта/синхронизации.
2. На кассе открыть `plugins.log`.
3. Подтвердить чтение новых значений в логах плагина:
   - `Config: ... ttl=..., pollDelay=..., pollTimeout=...`

Пример проверки в PowerShell:

```powershell
Select-String -Path "C:\path\to\plugins.log" -Pattern "\[SBGPay\] Config:" | Select-Object -Last 20
```

Артефакты:
- фрагмент `plugins.log` со свежими значениями
